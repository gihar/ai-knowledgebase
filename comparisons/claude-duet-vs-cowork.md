# Claude Duet vs Cowork

Claude Duet/Claude Code и Cowork можно понимать как два режима доступа Claude к среде разработки. По пересланной схеме, базовый агентный процесс одинаковый: понять задачу, изучить код, внести изменения, установить зависимости при необходимости, запустить проверки и вернуть результат. Различается не «интеллект» модели, а место выполнения действий и уровень изоляции.

## Duet / Claude Code

Duet, обозначенный на схеме как Claude Code в режиме `Code`, работает с прямым доступом к файлам проекта и локальной системе пользователя. Это даёт скорость и привычные интеграции: приложение, терминал, редакторы вроде VS Code и JetBrains, локальные зависимости и уже настроенное окружение.

Главный минус — риск: агент может читать, изменять или удалять то, к чему получил разрешение, запускать команды и устанавливать зависимости на пользовательской машине. Поэтому такой режим лучше подходит разработчикам, которые понимают последствия команд и могут контролировать права, diff и rollback.

## Cowork

Cowork на схеме описан как режим внутри приложения Claude, где задача выполняется через изолированную Linux VM-песочницу. Проект копируется в VM, окружение подготавливается отдельно, агент выполняет команды и тесты там, а обратно возвращаются только изменённые файлы и логи.

Зачем нужна эта прослойка:

- **безопасность:** потенциально вредоносный код не исполняется на основной машине;
- **изоляция:** зависимости и окружение не загрязняют систему пользователя;
- **контроль:** весь процесс проходит в управляемой VM;
- **откат:** проще сбросить окружение и начать заново.

Цена этой безопасности — дополнительная задержка на копирование проекта и старт VM, возможные сбои на длинных многошаговых задачах и повышенный расход токенов из-за подготовки окружения, логов и дополнительных наблюдений за VM.

## Практическая эвристика выбора

- Выбирать Duet/Claude Code, если важны скорость, локальная интеграция и пользователь готов контролировать права агента.
- Выбирать Cowork, если важнее безопасность, изоляция и простота для пользователя, который не хочет давать агенту прямой доступ к своей машине.
- Для [[agentic-coding-workflows]] Cowork — пример sandbox-first UX: агент остаётся тем же, но граница доверия переносится с локальной ОС в одноразовое или управляемое окружение.
