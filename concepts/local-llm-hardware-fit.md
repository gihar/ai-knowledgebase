# Подбор локальных LLM под железо

Подбор локальной LLM под железо — практический этап [[ai-engineering]], где модель выбирается не только по качеству, но и по RAM/VRAM, backend ускорения, quantization, контексту, ожидаемой скорости и категории задачи. Инструменты вроде [[llm-checker]] превращают этот этап из ручных экспериментов в предварительную оценку совместимости.

## Типовой workflow

1. Определить характеристики машины: CPU, GPU, RAM, backend ускорения, shared/dedicated VRAM.
2. Выбрать категорию задачи: coding, reasoning, multimodal, talking и т.п.
3. Получить shortlist моделей и конкретные команды загрузки, например через Ollama.
4. Проверить выбранную модель на реальных промптах, скорости, контексте и стабильности.
5. Зафиксировать рабочую конфигурацию для повторного использования в локальном ассистенте, Open WebUI, LM Studio или coding workflow.

## Почему это важно

Локальные LLM часто упираются не только в общий размер модели, но и в quantization, контекстное окно, shared memory на Apple Silicon, доступный GPU backend и параллелизм. Предварительная рекомендация снижает стоимость экспериментов: меньше скачиваний неподходящих моделей, меньше ложных выводов «модель плохая», когда проблема в неподходящем размере или настройках runtime.
